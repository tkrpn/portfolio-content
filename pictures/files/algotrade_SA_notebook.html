Project origin and short Summary¶

We decided to focus our project on the topic of financial news headline sentiment analysis. For that we used a fitting dataset from kaggle (https://www.kaggle.com/ankurzing/sentiment-analysis-for-financial-news/notebooks?sortBy=voteCount&group=everyone&pageSize=20&datasetId=622510)

The Dataset consists of roughly 5000 financial news headlines and provides a sentiment classification as a label (divided into "positive", "neutral", "negative"). Our goal with this project ist to train a machine learning classifier to analyse and recognise the sentiment of these news headlines.

Our motivation for this kind of project is the ever growing relevance NLP and Word-Tokenizing in the world of digital voice assistants and Automated Chatbots. A machine learning Model that can recognise the sentiment of the current news trends can also be very useful in the finance world for example if you are deciding about your future investments.

Removing disruptive Words¶

While testing our models we could observe that our dataset had some common words like "euro" , "mio" or "$" that would get very high feature importances but couldn't contribute much information for the sentiment of the news headline. We decided to drop these common disruptive words from our dataset, which proved successful and resulted in a better performing model.

In [17]:
def remove_synonyms(string, synonym_list):
    string_split = np.array(string.lower().split())
    for synonym in synonym_list:
        if synonym in string_split:
            mask = string_split == synonym
            string_split[mask] = ''
    
    return " ".join(string_split)

We also decided to drop all numeric values of our data, because our small dataset size wouldn't allow for the estimator to gain usefull information out of them.

In [18]:
def remove_numbers(string):
    tmp = re.sub(r'[.,]', '', string)
    tmp = re.sub(r'\d*[0-9]', '', tmp)
    return tmp

We are using lemmatizing and not stemming here, because lemmatizing is more accurate that stemming. That is because it uses an informed analysis of the word with its context in mind rather than just chopping off the end of the word like stemming.

    res = [lemmatizer.lemmatize(word) for word in tokenized_text]

Preelimiary Model Consideration¶

We chose two models to comapre on our data, which are Support Vector Machines (SVM) and RandomForest.

Support Vector Machines came to mind because we are using a Tf-Idf Vector to represent our String-Data, which have notoriously many features, especially when we are using an ngram-range to include wordpairs up to 3 words long. Kernelized SVMs are the perfect fit for that, because they can be computed effectively with a large number of features if the sample size isn't too high.

We chose Random Forests to compare. It is an Ensemble Method that can also work with a large number of features because it only uses a random subset of them to train one of its Decision-Trees.

Defining a function for nested cross-validation¶

In [30]:
from sklearn.model_selection import KFold, cross_val_score
from sklearn.metrics import roc_auc_score

def nestedCV(estimator, parameter, X, y, verbose_level=5):
    inner_cv = KFold(n_splits=4, shuffle=True, random_state=42)
    outer_cv = KFold(n_splits=4, shuffle=True, random_state=42)

    # Non_nested parameter search and scoring 
    clf = GridSearchCV(estimator=estimator, param_grid=parameter, cv=inner_cv, verbose=verbose_level, n_jobs=-1, scoring='roc_auc_ovo') # -1 for paralellizing
    clf.fit(X, y)

    # Nested CV with parameter optimization
    nested_score = cross_val_score(clf, X=X, y=y, cv=outer_cv)
    return nested_score.mean(), clf

def confusion_matrix(estimator, ax, test_size=.25, classifier_title=''):

    X_train, X_test, y_train, y_test = train_test_split(news_headline_data['text_final'], news_headline_data['res'], test_size=test_size, random_state=42)

    estimator.fit(X_train, y_train)

    class_names = ['negative', 'neutral', 'positive']

    titles_options = [(f"Confusion matrix, without normalization [{classifier_title}]", None, ax[0]),
                      (f"Normalized confusion matrix [{classifier_title}]", 'true', ax[1])]
    for title, normalize, axis in titles_options:
        disp = plot_confusion_matrix(estimator, X_test, y_test,
                                     display_labels=class_names,
                                     ax=axis,
                                     cmap=plt.cm.Blues,
                                     normalize=normalize)
        disp.ax_.set_title(title)
    
    y_pred = estimator.predict(X_test)
    print(f"F1-score [{classifier_title}]: {f1_score(y_test, y_pred, average='micro')}")
    
    return disp

def plot_confusion_matrices(svc_estimator, rf_estimator):

def roc_curves_multi_class(estimator, axis, plot_micro=True, plot_macro=False, title='AUC Curves', test_size=.25):

    X = news_headline_data['text_final']
    y = news_headline_data['res']
    y = label_binarize(y, classes=['negative', 'neutral', 'positive'])
    
    n_classes = 3

    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=test_size, random_state=0)

    classifier = OneVsRestClassifier(estimator)
    
    y_score = classifier.fit(X_train, y_train).predict_proba(X_test)
    
    # Compute ROC curve and ROC area for each class
    fpr = dict()
    tpr = dict()
    roc_auc = dict()
    for i in range(n_classes):
        fpr[i], tpr[i], _ = roc_curve(y_test[:, i], y_score[:, i])
        roc_auc[i] = auc(fpr[i], tpr[i])

    # Compute micro-average ROC curve and ROC area
    fpr["micro"], tpr["micro"], _ = roc_curve(y_test.ravel(), y_score.ravel())
    roc_auc["micro"] = auc(fpr["micro"], tpr["micro"])

    # First aggregate all false positive rates
    all_fpr = np.unique(np.concatenate([fpr[i] for i in range(n_classes)]))

    # Then interpolate all ROC curves at this points
    mean_tpr = np.zeros_like(all_fpr)
    for i in range(n_classes):
        mean_tpr += interp(all_fpr, fpr[i], tpr[i])

    # Finally average it and compute AUC
    mean_tpr /= n_classes

    fpr["macro"] = all_fpr
    tpr["macro"] = mean_tpr
    roc_auc["macro"] = auc(fpr["macro"], tpr["macro"])

    lw=2

    if plot_micro:
        # Plot Micro-Average ROC Curve
        axis.plot(fpr["micro"], tpr["micro"],
                 label='micro-average ROC curve (area = {0:0.2f})'
                       ''.format(roc_auc["micro"]),
                 color='deeppink', linestyle=':', linewidth=4)

    if plot_macro:
        # Plot Macro-Average ROC Curve
        axis.plot(fpr["macro"], tpr["macro"],
                 label='macro-average ROC curve (area = {0:0.2f})'
                       ''.format(roc_auc["macro"]),
                 color='navy', linestyle=':', linewidth=4)

    axis.plot([0, 1], [0, 1], 'k--', lw=lw)
    axis.set_xlim([0.0, 1.0])
    axis.set_ylim([0.0, 1.05])
    axis.set_xlabel('False Positive Rate')
    axis.set_ylabel('True Positive Rate')
    axis.set_title(title)
    axis.legend(loc="lower right")
    
    return [fpr['micro'], tpr['micro'], roc_auc['micro']], [fpr['macro'], tpr['macro'], roc_auc['macro']]

def plot_multi_roc(roc_arr, titles, axis, plot_title='ROC Curve Comparison'):
    lw = 2
    # roc_arr is a list of fprs, tprs and auc_score for each roc curve
    colors = ['red', 'green']
    for i in range(len(roc_arr)):
        axis.plot(roc_arr[i][0], roc_arr[i][1],
                     label=f'[{titles[i]}] micro-average ROC curve (area = {0:0.2f})'
                           ''.format(roc_arr[i][2]),
                     color=colors[i%len(colors)], linestyle=':', linewidth=4)
    
    axis.plot([0, 1], [0, 1], 'k--', lw=lw)
    axis.set_xlim([0.0, 1.0])
    axis.set_ylim([0.0, 1.05])
    axis.set_xlabel('False Positive Rate')
    axis.set_ylabel('True Positive Rate')
    axis.set_title(plot_title)
    axis.legend(loc="lower right")
